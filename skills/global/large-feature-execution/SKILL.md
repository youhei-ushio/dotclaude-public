# 大型機能の実行（永続プラン + subagent 駆動）

Plan モードで合意した実装方針を、**長時間・context 圧縮を跨いで**遂行するための運用。
**大型機能のときだけ**使う（小規模は通常の Plan モードで十分。過剰適用しない）。

## いつ使うか（適用条件）

- タスクが複数（概ね 4 つ以上）に分割でき、1 セッション内で終わらない見込み。
- 途中で context が圧縮され、Plan モードの揮発した方針を失うリスクがある。
- 上記に当てはまらないなら、通常の Plan モード + そのまま実装で十分。本 skill は使わない。

## 1. プランの永続化

Plan モードで承認された方針を、揮発する前にファイルへ保存する。

- 保存先: `docs/development/plans/{ISO日付}-{内容}.md`（`docs/development/` 配下の本 skill 推奨パス。一般的な命名・配置規約は `documentation-standards` に従う）。
- 内容: 背景（なぜ）/ タスク分割（順序と依存）/ 各タスクの完了条件 / 検証方法。
- これにより context 圧縮や別セッションを跨いでも、ファイルを読めば方針を復元できる
  （harness の Plan モード / `~/.claude/plans/` は揮発しうるため、リポ管理下に残すのが本運用の要点）。
- プランは生きた文書として、タスク完了ごとに進捗（done / 次タスク）を更新する。

## 2. タスク単位の subagent 駆動

プランのタスクを 1 つずつ subagent に渡して実装する（CLAUDE.md「サブエージェント 1 つにつき 1 タスク」）。

- **1 subagent = 1 タスク**。タスクの目的・完了条件・触ってよい範囲を明示して渡す。
- 並行で複数タスクを走らせて衝突しうるなら `isolation: "worktree"` で分離する
  （`review-pr` が採る worktree 分離パターンと同じ。親の作業ツリー・ライブ設定を守る）。
- 各 subagent には親セッションの冗長な文脈でなく、そのタスクに必要な情報だけを渡す（context 節約）。

## 3. タスク間レビューゲート

タスクを 1 つ仕上げるたびに、次へ進む前にレビューを挟む。

- 完了条件を満たしたか、プランから逸脱していないかを確認する。
- コード変更を伴うタスクは `review-pr` のセルフレビューに載せる（独立 subagent によるレビュー）。
- ゲートを通らなければ次タスクに進まない。問題があればプランを更新してから続行する。

## Plan モードとの関係

- 本 skill は Plan モードを**置換しない**。Plan モードで方針を立て承認を得る流れはそのまま。
- 本 skill が足すのは「承認後の方針をファイルに残す」「実装をタスク単位 subagent + レビューゲートで
  回す」の 2 点だけ。大型機能で方針が揮発・散逸するのを防ぐための延長運用。
