# 作業引き継ぎ（handoff）スキル

作業状態をファイルに保存し、後で読み込んで続きから再開できるようにするスキル。
プロジェクトをまたいでグローバルに利用する。

## 設定

- **保存先ディレクトリ**: `~/.claude/handoff/`（存在しなければ初回 save 時に自動作成する）
- **ファイル名フォーマット**: `YYYY-MM-DD-HHMMSS-{サニタイズ後タスク名}.md`
- **テンプレート**: `~/.claude/skills/global/handoff/template.md`
- **タスク名のサニタイズ規則**（ファイル名に使う際に必ず適用する。frontmatter の `task` には元のタスク名を保存する）:
  1. 不正文字（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`, 改行）はハイフンに置換する
  2. 空白文字（半角スペース・全角スペース・タブ）はハイフンに置換する
  3. 連続するハイフンは 1 個に圧縮する
  4. 先頭・末尾のハイフンは削除する
  5. 日本語はそのまま使ってよい
- **`title` skill 不在時の動作**: 本 skill は `title` skill（`~/.claude/skills/title/`）と任意連携するが、`title` skill や `~/.claude/tasks/` が存在しない環境でも単体で動作する。連携箇所（save のデフォルトタスク名取得 / load のタスク名自動設定）は、対象が無ければ静かにスキップして処理を続行する。

## 動作モード

引数の最初のトークンで分岐する（`save` / `load` / `list`）。
引数が無い、もしくは不明な場合は使い方を表示する。

### `save [タスク名]` — 現在の作業状態を保存

1. タスク名を決定する（優先順位）:
   1. 引数で明示的に渡されたタスク名
   2. `~/.claude/tasks/{tty_id}.txt` に設定済みのタスク名（title スキルで設定されたもの）
   3. 会話の文脈から推測したタスク名
   - **優先順位 2 の fallback**: `~/.claude/tasks/{tty_id}.txt` やディレクトリ自体が無い場合は、警告を出さず静かに優先順位 3（会話文脈から推測）に fallback する。`title` skill 未導入の環境では `~/.claude/tasks/` 自体が存在しないことが多い。
   - **title の値と会話文脈が明らかに食い違う場合**（例: title は別タスクの残骸で、現在の会話は別テーマを扱っている）は、**どちらを採用するかユーザーに短く確認すること**。勝手に保存しない。
2. `mkdir -p ~/.claude/handoff` でディレクトリを確保
3. **保存方針と引き継ぎ値を決定する**（この時点ではまだファイルを作成・削除しない）:
   - **同一タスク名の既存ファイルを検索**: `~/.claude/handoff/` 内で、ファイル名がサニタイズ後タスク名で終わるファイルを Glob で探す。pattern 例: `~/.claude/handoff/*-{サニタイズ後タスク名}.md`
   - **既存ファイルが複数ヒットした場合**: ファイル名のタイムスタンプ降順で最新を上書き対象候補とし、対象ファイル名をユーザーに提示して上書き可否を確認する（勝手に複数を消さない）。
   - **既存ファイルが見つかった場合**: ユーザーに「既存の handoff（{ファイル名}）を上書きしますか？ 新規作成しますか？」と確認する
     - **上書き**: 旧ファイルの frontmatter から `created` を読み取って保持し、`updated` は現在時刻にする。旧ファイル名は「新ファイル書き出し成功後に削除する対象」として控えるだけにする（ここでは削除しない）。
       - 旧ファイルに `created` が無い／パースできない場合は、`created` に現在時刻を代入する（フォールバック）。
     - **新規作成**: `created` と `updated` を両方とも現在時刻に設定し、別ファイルとして作成する（旧ファイルは削除しない）。
   - **既存ファイルが無い場合**: `created` と `updated` を両方とも現在時刻に設定する。
4. 現在時刻を取得: `date +%Y-%m-%d-%H%M%S`
5. 新ファイルパスを決定: `~/.claude/handoff/{YYYY-MM-DD-HHMMSS}-{サニタイズ後タスク名}.md`
6. テンプレート `~/.claude/skills/global/handoff/template.md` を Read し、プレースホルダに**現時点の会話文脈から分かる内容**を埋めて新ファイルに Write する
   - `{...}` で囲まれたプレースホルダ・ガイダンス文（例: `{今どこまで進んだか…}`）は最終出力には残さず、実内容で置き換える。埋める内容が無い項目は `-` のみの空リストにする（後から手で追記できるように）
   - frontmatter は `template.md` 冒頭の `---` 行から始める。テンプレートにフェンス（` ``` `）は含まれていないので、そのまま書き出してよい
   - 事実に基づき、推測が入る場合は `（推測）` と明示する
7. **新ファイルの書き出し成功を確認してから**、手順 3 で控えた上書き対象の旧ファイルがあれば削除する（アトミック性の担保。書き出し前に旧ファイルを消さない）
8. 書き出したファイルパスを報告する

### `load [番号]` — 保存済みhandoffから再開

1. `~/.claude/handoff/` 内の `*.md` をファイル名降順（＝最終更新が新しい順）で列挙する
2. ファイルが1つも無ければ「handoffはありません」と報告して終了
3. 引数で番号が指定されていればその番号のファイルを使う。未指定なら以下のように一覧を提示してユーザーに番号を尋ねる:
   ```
   handoffを選んでください:
     1. 2026-05-26-140000 | タスクA（作成: 2026-04-24）
     2. 2026-04-23-101530 | タスクB
     ...
   ```
   - ファイル名のタイムスタンプ（＝最終更新日時）を表示する
   - frontmatter の `created` と `updated` が異なる場合は `（作成: YYYY-MM-DD）` を末尾に付けて、元の作成日もわかるようにする
   （番号選択はユーザーの返答を待つ。`load <番号>` で直接指定されていればスキップ）
4. 選ばれたファイルを Read で読み込み、内容を**そのまま提示**する（提示時の省略ルールは「注意事項」を参照）
5. frontmatter の `task` を現在のTTYのタスク名として**自動設定する**（`title` skill と連携するための任意処理。失敗しても load 自体は成功扱い）:
   - TTY ID を取得する: `tty` コマンドの出力（例: `/dev/ttys003`）から `basename` でデバイス名（`ttys003`）を取り出して ID とする。`title` skill が導入済みなら、その識別方法に揃えるとなお良い。
   - `tty` が `not a tty` 等で ID を取得できない場合は、この手順 5 全体をスキップし「タスク名の自動設定はスキップしました（TTY 未取得）」と一言だけ表示する。
   - `mkdir -p ~/.claude/tasks` でディレクトリを確保
   - frontmatter の `task` の値を `~/.claude/tasks/{tty_id}.txt` に書き出す
   - 書き出した内容を「タスクを「{タスク名}」に設定しました。」と一言報告する
6. 続けて「このhandoffの続きから作業を再開します。次のアクションから着手しますか？」と一言添える

### `list` — 保存済みhandoff一覧を表示

1. `~/.claude/handoff/` 内の `*.md` をファイル名降順（＝最終更新が新しい順）で列挙する
2. 各ファイルについて以下を表示:
   - 番号
   - 最終更新日時（ファイル名から抽出）
   - タスク名（frontmatter の `task` またはファイル名から抽出）
   - frontmatter の `created` と `updated` が異なる場合は `（作成: YYYY-MM-DD）` を末尾に付ける
3. 件数が0なら「handoffはありません」と報告

### 引数なし or 不明

使い方を表示:

```
/handoff save [タスク名]  - 現在の作業を保存
/handoff load [番号]      - 保存済みhandoffから再開
/handoff list             - handoff一覧を表示
```

## 注意事項

- **このスキルは `memo` / `task` スキルとは連携しない**。`title` スキルとは以下の点で連携する:
  - `save` 時: タスク名のデフォルト値として title の保存ファイル（`~/.claude/tasks/{tty_id}.txt`）を参照する
  - `load` 時: 読み込んだ handoff の `task` を現在のTTYのタスク名として自動設定する
- `save` 時は会話コンテキストから積極的に情報を抽出して埋めること。空のテンプレートだけ書き出すのは避ける
- **機微情報は保存しない**: API キー・認証トークン・パスワード・秘密鍵などが会話に含まれていても handoff ファイルには書き出さず、`(redacted)` に置き換える（handoff は平文で `~/.claude/handoff/` に永続化されるため）
- テンプレート内の `{...}` プレースホルダ・ガイダンス文は最終出力には残さず、必ず実内容で置き換える
- `load` で内容を提示する際は、マークダウンの構造を保ったまま全文を出す（要約しない）。ただし極端に長い handoff（目安 200 行超）は frontmatter と主要セクションを全文提示し、残りは「（…以下省略。全文はファイル参照）」と省略してコンテキスト枯渇を避ける
- ファイル名に使うタスク名は**必ず**「設定」のサニタイズ規則を適用する。サニタイズ前後で見た目が大きく変わる場合は、frontmatter の `task` に元のタスク名を保存しておくこと
